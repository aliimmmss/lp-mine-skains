# Status document ownership

When a milestone or safety gate changes, update these files together:

1. `README.md` for the concise public summary.
2. `docs/CURRENT_STATUS.md` for the explicit implemented and non-authorized boundary.
3. `docs/STATUS_CHANGELOG.md` for the dated change record.
4. `docs/NEXT_M4_SLICE.md` only when the separately reviewed next gate changes.

Safety claims must remain fail-closed and must not imply authorization for wallet access, signing, submission, money movement, or capital deployment.
